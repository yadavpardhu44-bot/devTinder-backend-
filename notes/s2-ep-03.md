## Creating Our Express Server

# 📌 Episode Summary

In this episode, we moved from the built-in `http` module to **Express.js**, the most popular web framework for Node.js.

Instead of writing long boilerplate code using `http.createServer()`, Express allows us to create servers, routes, and middleware with much simpler syntax.

We also learned how npm manages dependencies, why `package.json` and `package-lock.json` are important, how to install packages locally, and how to use **nodemon** for automatic server restart during development.

```javascript

```

```javascript

```

```javascript

});
```

# Understanding Important Files

## package.json

Project information.

Contains

* project name
* scripts
* dependencies
* version

Think of it as the **identity card** of your project.

---

## package-lock.json

Stores the exact versions of every installed package.

Example

Even if Express depends on 50 other packages, every version is recorded.

Purpose

* Same versions on every machine
* Faster installation
* Reproducible builds

Never edit manually.

---

## node_modules

Contains the actual installed packages.

Very large folder.

Never push it to Git.

Add it to

```
.gitignore
```

## 9. What are Dependencies?

Dependencies are external packages that your project **needs to run correctly**.

Examples:

* Express
* Mongoose
* React
* Axios

They are installed using npm and stored in the `dependencies` section of `package.json`.

Example:

```json
"dependencies": {
    "express": "^5.1.0"
}
```

---

## 10. What is the use of `-g` while `npm install`?

`-g` means **global installation**.

Example:

```bash
npm install -g nodemon
```

Installed once on your computer and available from any project.

Without `-g`:

```bash
npm install nodemon
```

Installed only inside the current project.

### When to use global installation?

Use `-g` for command-line tools you want available everywhere, such as:

* nodemon
* npm
* typescript (`tsc`)

Most project dependencies (Express, React, Mongoose, etc.) should be installed **locally**, not globally.

## 11. Difference between Caret (`^`) and Tilde (`~`)

Suppose the installed version is:

```
4.18.2
```

### Caret (`^`)

```json
"express": "^4.18.2"
```

Allows updates that do **not** change the major version.

Examples allowed:

```
4.18.3 ✅
4.19.0 ✅
4.20.5 ✅
```

Not allowed:

```
5.0.0 ❌
```

**Rule:** `^` allows **minor + patch** updates.

---

### Tilde (`~`)

```json
"express": "~4.18.2"
```

Allows only **patch** updates.

Examples allowed:

```
4.18.3 ✅
4.18.4 ✅
```

Not allowed:

```
4.19.0 ❌
5.0.0 ❌
```

**Rule:** `~` allows **patch** updates only.

## 💡 Interview Tips

* Prefer installing project libraries (like Express) **locally**.
* Commit both `package.json` and `package-lock.json` to Git, but **never** commit `node_modules`.
* Use `nodemon` as a **dev dependency** during development.
* Remember the versioning rule:

  * **`^` = Minor + Patch updates**
  * **`~` = Patch updates only**
